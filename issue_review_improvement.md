## 📌 상세 설명
> 어떤 동작을 기대하는지, 왜 필요한지 설명해주세요.

### 📂 도메인
> 개인 담당 도메인(파트) 표시 후 작성해주세요.
> ex) Dashboard / Host / Booking ...
- **MyPage / Review**

- **현재 상황:**
  - 리뷰 태그 데이터가 체계적으로 관리되지 않아 UI 표현에 어려움이 있음.
  - 리뷰 수정 기능이 예약 내역 페이지에 있어 접근성이 떨어지고, 삭제 기능이 부재함.
  - 리뷰 수정 시 체크인/체크아웃 정보와 숙소 이미지가 누락되는 문제 발생.

- **개선 내용:**
  - `review_tag` 테이블에 40개 표준 태그 데이터를 구축하고 API로 제공.
  - 프론트엔드에서 태그 ID기반으로 카테고리를 분류하여 직관적인 선택 UI 제공.
  - 리뷰 내역 페이지(`ReviewHistoryView`)로 수정/삭제 기능을 통합하고 UI 정보(이미지, 날짜) 보강.

- **기대 효과:**
  - 리뷰 데이터(태그)의 표준화 및 관리 용이성 확보.
  - 사용자가 본인의 리뷰를 한곳에서 쉽고 편리하게 관리(조회/수정/삭제) 가능.
